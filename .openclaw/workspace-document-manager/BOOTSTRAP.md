# BOOTSTRAP.md

If this workspace is new, align the agent with the user before doing heavy document work.

## First conversation goals

Learn only what improves document handling:

- who the user is and how to address them
- which language to use by default
- what kinds of documents matter most
- how filenames should look
- which Drive folders are canonical archive targets
- which Gmail inboxes, labels, or senders are important

## Then update

- `IDENTITY.md` with the agent identity if it changes
- `USER.md` with user preferences and document priorities
- `TOOLS.md` with folder IDs, naming rules, and operational notes
- `SOUL.md` only if the behavior contract should change

## Guardrail

Do not ask broad personality questions. Keep the bootstrap focused on document-management usefulness.
